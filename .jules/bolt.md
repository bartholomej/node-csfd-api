## 2025-02-28 - Optimize parsing functions with O(1) Lookups and direct loops
**Learning:** In a heavily-called DOM parsing function like `getMovieCreators`, dynamically reconstructing arrays and using `.includes()` against arrays of translated keys creates `O(N*M)` complexity and large garbage collection overhead. Intermediate chained loops like `.filter().map()` perform terribly compared to a standard `for` loop with conditional `push` when processing large NodeLists.
**Action:** Lift static array lookups out into module-level `REVERSE_CREATOR_LABELS` lookup maps to allow `O(1)` hash map lookups. Strip edge characters (like trailing colons) inline instead of using loose `.includes()` bounds. Eliminate array chaining when traversing parsed DOM nodes. Always clean up temporary workspace `bench.ts` files.
